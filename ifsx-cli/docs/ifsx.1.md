# NAME

ifsx -- IFS Extract/Repack Tool for QNX images

# SYNOPSIS

**ifsx** *command* [**-h**] [*arguments*]

**ifsx extract** [**--pre-extract** *hook*]... [**--post-extract** *hook*]... *image.ifs* *output-dir*

**ifsx pack** [**--pre-pack** *hook*]... [**--post-pack** *hook*]... *extracted-dir* [*output.ifs*]

# DESCRIPTION

**ifsx** extracts and repacks QNX IFS boot images.
It is a wrapper around the **dumpifs**(1) and **mkifs**(1) tools from the
QNX Software Development Platform (SDP).
Both tools must be present in **PATH** before invoking **ifsx** — source the
QNX SDP environment script first (e.g. **source ~/qnx800/qnxsdp-env.sh**).

At startup **ifsx** creates **~/.ifsx/** and its four subdirectories
(**pre-extract**, **post-extract**, **pre-pack**, **post-pack**) if they do
not already exist. Executable files placed in those directories are available
as *hooks* (see **HOOKS** below).

# COMMANDS

**extract** (alias: **x**)
:   Extract an IFS image to a self-contained directory.
    Produces a **_ifsx.build** buildfile and a **_ifsx.meta** metadata file
    alongside the extracted filesystem tree.

**pack** (alias: **p**)
:   Repack an extracted directory back into an IFS image.
    The directory must contain the **_ifsx.build** file generated by
    **extract**.

# OPTIONS

## extract

**ifsx extract** [**--pre-extract** *hook*]... [**--post-extract** *hook*]... *image.ifs* *output-dir*

*image.ifs*
:   Path to the IFS image to extract.

*output-dir*
:   Directory where the extracted files will be written.
    The directory is created if it does not exist.

**--pre-extract** *hook*
:   Name of an executable in **~/.ifsx/pre-extract/** to run before extraction.
    May be repeated; hooks are executed in the order given on the command line.
    Each hook receives *image.ifs* and *output-dir* as positional arguments.
    A non-zero exit code aborts the operation.

**--post-extract** *hook*
:   Name of an executable in **~/.ifsx/post-extract/** to run after extraction.
    Same semantics as **--pre-extract**.

## pack

**ifsx pack** [**--pre-pack** *hook*]... [**--post-pack** *hook*]... *extracted-dir* [*output.ifs*]

*extracted-dir*
:   Path to the directory previously created by **ifsx extract**.
    Must contain **_ifsx.build**.

*output.ifs*
:   Path for the output IFS image.
    Defaults to *extracted-dir* with **.ifs** appended.

**--pre-pack** *hook*
:   Name of an executable in **~/.ifsx/pre-pack/** to run before packing.
    May be repeated; hooks are executed in the order given on the command line.
    Each hook receives *output.ifs* and *extracted-dir* as positional arguments.
    A non-zero exit code aborts the operation.

**--post-pack** *hook*
:   Name of an executable in **~/.ifsx/post-pack/** to run after packing.
    Same semantics as **--pre-pack**.

# HOOKS

**ifsx** supports user-defined hook executables stored under **~/.ifsx/**:

    ~/.ifsx/
        pre-extract/    run before extraction
        post-extract/   run after extraction
        pre-pack/       run before packing
        post-pack/      run after packing

Any executable file placed in one of these directories becomes a hook.
Hooks are called with two positional arguments:

1. The IFS image path (absolute)
2. The directory path (absolute)

For **extract** hooks both arguments refer to the source image and output
directory.  For **pack** hooks the first argument is the output IFS path and
the second is the extracted directory.

If a hook exits with a non-zero code the operation is aborted immediately and
the remaining hooks in the chain are not executed.
Both stdout and stderr of the hook are printed to the terminal.

# ENVIRONMENT

**PATH**
:   Must include the directory containing **dumpifs** and **mkifs**.
    If either tool is not found, **ifsx** will print a warning at startup:
    *Warning: dumpifs and/or mkifs not found. Did you source the SDP?*

# EXAMPLES

Extract an IFS image:

    ifsx extract image.ifs output/

Extract, running hooks before and after:

    ifsx extract --pre-extract validate --post-extract sign image.ifs output/

Repack with an explicit output path:

    ifsx pack output/ new_image.ifs

Repack using the default output name (*output.ifs*):

    ifsx pack output/

Repack with a post-pack hook:

    ifsx pack --post-pack deploy output/ new_image.ifs
